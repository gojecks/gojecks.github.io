jEliDB

jEliDb or jDB is an open source JavaScript library that provides powerful in-memory database capabilities to both browser and server applications.

Introduction

JavaScript object literal look a lot like database table records, And wrapping a group of javascript objects up in an array gives you something that looks a lot like a database table.

jDB is created out of Javascript Objects to produce result with a uniform and familiar SQL-like interface.
Using jDB in your application help reduces development time, improves performance, simplifies maintenance, and increases quality.

Why use jDB

  • Extremely fast
  • Powerful JavaScript-centric data selection engine
  • SQL inspired features such as insert, update, unique, count, and more
  • Import and export data (JSON,csv and html) format
  • Synchronize and replicate Data on the fly using jSync Algorithm
  • cross browser support
  • Easy to extend with your own functions
  • Compatible with any DOM library
  • Access your DB anywhere in the world

Insert Data into Table

Inserting Data into a table requires a transaction to be performed. Transaction FN requires two parameter TBL_NAME (required) , MODE (Optional - "readonly,writeonly or readwrite")
Insert accepts ARRAY of OBJECTS or ARRAY of ARRAYS. Column name must be defined when using ARRAY of Objects, while for ARRAY of ARRAYS the data must follow according to their column position in the table.
In Array of Objects AUTO_INCREMENT field can be omitted, while for ARRAY of Arrays cannot.

Inserting Data with Array of Objects

[{fullName:"Foo Bar",email:"foo@bar.com"},{fullName:"Bar Foo",email:"bar@foo.com"}] -members

ID field was omitted, because it has a configuration of AUTO_INCREMENT

Inserting Data with Array of Arrays

[[1,"Foo Bar","foo@bar.com"],[2,"Bar Foo","bar@foo.com"]]
As said above, the ID field cannot be omitted even if having the auto_incremement configuration when inserting data with ARRAY of ARRAYS. Data in each array will be Mapped to the columns
Note : If "MODE" is undefined transaction is treated as read_only, for insert we re using the writeonly mode.

Select From Table

Records can be fetched from table using the "SELECT" function, this also requires a transaction to be performed.
  • * - To select all columns
  • -columnName : Select a particular column set from the table ( Seperate multiple columns with a comma)
  • -columnName as user-defined : select columns and return as user-defined Name (eg : -fullName as fName)

Using CLAUSE in Select

  • WHERE :

    eg : where(id === 1)

  • -LIKE : -LIKE:expression

    eg where(fullName LIKE:foo)

  • Join Clause :

    List of Join Clause to use

    • INNER
    • LEFT
    • RIGHT
    • OUTER
  • Limit:

    eg: limit(0,10)

//Below example selects record from 1 table
db
.transaction(TBL_NAME)
.select('name,email,password,id')
.execute()
.onError(errorHandler)
.onSuccess(function(e)
{
  console.log("Success Result:",e.getResult())
});

//Below example selects record from 2 table using the join CLAUSE
  db
  .transaction([TBL_1,TBL_2])
  .select('TBL_1.COL_NAME,TBL_2.COL_COURSE')
  .join('inner:TBL_2')
  .on('TBL_1.course = TBL_2.id')
  .execute()
  .onError(errorHandler)
  .onSuccess(function(e)
  {
    console.log("Success Result:",e.getResult())
  });

  //using WHERE and LIKE CLAUSE
   db
  .transaction(TBL_NAME)
  .select('name,email,password,id')
  .where("COL_NAME like:foo")
  .execute()
  .onError(errorHandler)
  .onSuccess(function(e)
  {
    console.log("Success Result:",e.getResult())
  });

//using Limit Clause
db
  .transaction(TBL_NAME)
  .select('COL_1,COL_2,COL_3')
  .where("COL_NAME like:foo")
  .limit('0,10')
  .execute()
  .onError(errorHandler)
  .onSuccess(function(e)
  {
    console.log("Success Result:",e.getResult())
  });

Truncate Table

Table can be Truncate using the Truncate function. Accepts Parameter Flag : YES or NO.
NO flag will trigger an error
//Below example truncate our table
db
.table(TBL_NAME)
.onSuccess(function(trun)
{
  var flag = YES;
  trun.result.truncate(flag);
});

Drop Table / Database

Table can be dropped using the Drop function. Accepts Parameter Flag : YES or NO.

Note : NO flag will trigger an error

//Below example drops our table
  db
  .table(TBL_NAME)
  .onSuccess(function(tbl){
    var flag = YES;
    tbl.result.drop(flag);
  })
  .onError(function(e){
    result = e;
  })

Synchronize Database or Table

Synchronize changes on the client to the server, we created a tool that can compare and check for changes between client and server if the server have the latest changes, the client will be updated with the server records else changes will be pushed to server for update

db
.synchronize()
.Entity([TBL_1]) // Array of TABLES to sync
.configSync({}) //Configuration used in sync method
.processEntity()

Options for configuration

  • serviceHost

    Define the location of your server, by default is null e.g http://localhost:9000/jDB/

  • conflictResolver

    Define a function that will resolve conflict, this function is only initialized when there is a conflict between server and client tables.

    Parameter: serverData, clientData

    Only define a function if you don't want our tool to resolve the conflict.

  • logService

    Define a function that can register all logs thrown from the sync method.

    Parameter: message

  • handler

    • onSuccess

      Define a function that will perform an action when sync is successfull

    • onError

      Define a function that will perform an action when sync fails.

This configurations can also be define when creating the database and it's extensible. Please look at Create a database

Using jCMD

One of the best feature of jDB is, it's ability to perform query using its own query Syntax.

Type -help to see command syntax


Create a table : Syntax : -CREATE -TABLE_NAME -COLUMNS

-CREATE -members -[{ID:{type:"INTERGER",AUTO_INCREMENT:1},email:{type:"VARCHAR"}}]

COLUMNS command is an ARRAY OF OBJECTS with list of columns with their configuration. A column must contain a type of input to accept. For empty column pass an empty ARRAY : []

Alter Table Syntax : -ALTER -TABLE_NAME

Add New Column :

-add -new -column -COLUMN_NAME -CONFIGURATION

Lets add a new column to our table

ALTER -members -add -new -column -fullName -{type:"VARCHAR",defaultValue:"Foo Bar"}

Configuration must be a javascript Object

Adding Primary Key Syntax :-add -primary -key -column_name

Drop a column Syntax : -drop -column_name
Column dropped will be removed from table rows.
Insert Data with query : Syntax : -insert -data -table_name
-data command accepts ARRAY of OBJECTS or ARRAY of ARRAYS. Column name must be defined when using ARRAY of Objects, while for ARRAY of ARRAYS the data must follow according to their column position in the table.
In Array of Objects AUTO_INCREMENT field can be omitted, while for ARRAY of Arrays cannot.

Inserting Data with Array of Objects into our members table

-insert -[{fullName:"Foo Bar",email:"foo@bar.com"},{fullName:"Bar Foo",email:"bar@foo.com"}] -members

ID field was omitted, because it has a configuration of AUTO_INCREMENT

Inserting Data with Array of Arrays into our members table

-insert -[[1,"Foo Bar","foo@bar.com"],[2,"Bar Foo","bar@foo.com"]] -members
As said above, the ID field cannot be omitted even if having the auto_incremement configuration when inserting data with ARRAY of ARRAYS. Data in each array will be Mapped to the columns
Select From Table Syntax : -select -columns -table_name

-columns command accepts different parameter

  • * - To select all columns
  • -columnName : Select a particular column set from the table ( Seperate multiple columns with a comma)
  • -columnName as user-defined : select columns and return as user-defined Name (eg : -fullName as fName)

Using CLAUSE in Select

  • WHERE : -Where -expression

    eg : -select -* -members -where -(id === 1)

  • -LIKE : -LIKE:expression

    eg : -select -* -members -where -fullName -LIKE:foo

  • Join Clause : -join -CLAUSENAME:tableName -on -expression List of Join Clause to use
    • INNER
    • LEFT
    • RIGHT
    • OUTER

    eg : -select -members.fullName,info.address -members -join -inner:info -on -memebers.ID === info.PID

Updating records : Syntax : -update -tableName -data -WHERE -expression
Data command accepts only 2 type of data OBJECT and params
  • OBJECT : {fullName:"Foo Boo",email:"foo@boo.com"}
  • PARAM : fullName ="Foo Boo",email="foo@boo.com"

eg : -update -members -fullName="Foo Boo" -where -(ID === 1 || fullName ==="Foo Bar")

Deleting records : Syntax : -delete -tableName -WHERE -expression

eg : -delete -memebers -where -(ID === 1 || fullName ==="Foo Bar")

Truncate a table : -truncate -tableName -flag(yes or no)

eg: -truncate -members -yes

Truncating a table deletes every record in the table, flag must be set to yes before task is performed, else an error will be thrown.
Dropping a table Syntax : -drop -table -tableName -flag(yes or no)

eg: -drop -table -members -yes

Dropping a table removes the table from the Database, flag must be set to yes before task is performed, else an error will be thrown.
Importing records : Syntax : -import -tableName

Importing record opens a dialog box to select the file you want to import to the table, accepted files are (.csv,.html,.json)

if the table doesn't exist it will be created and data will be inserted into the table.
Exporting records : Syntax : -export -tableName -format -task

Task List

  • Download : -d -fileName
  • Print : -p to console
Accepted types are (.csv,.html,.json), title is optional and can be left blank
Data Synchronization Syntax : -sync -tableName
Without definig a tableName, all tables in the database will be synchronized. Table Name must be in ARRAY format

eg : -sync -["members"] or -sync
