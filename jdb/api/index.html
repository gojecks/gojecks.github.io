# jEli.$jDB(DB_NAME,DB_VERSION)

  • Arguments

    • {string} DB_NAME (required)
    • {string|Number} DB_VERSION

  • Return

    {object} jDB Instance

  • Usage

    var DB; 
    var dbInstance = new jEli.$jDB(DB_NAME,DB_VERSION)

  • # isClient()

    • Details

      Initialize isClient() method to set the env to production. default: dev

  • # requiresLogin()

    • Details

      Initialize requiresLogin() method to set the login flag for a database. default: false

  • # open([definition])

    • Arguments

      • {object} definition

    • return: jDB Promise Instance

    • Details

      Definition

      • Storage: [localstorage, sessionstorage, websql, indexedDB] (default:localStorage)

      • Live:{boolean} (default:false)

        Set to true for auto-sync

      • serviceHost

        Define the location of your server, by default is null e.g http://ws.jdb.bezynet.com

      • conflictResolver

        Define a function that will resolve conflict, this function is only initialized when there is a conflict between server and client tables.

        Parameter: serverData, clientData

        Only define a function if you don't want our tool to resolve the conflict.

      • logService

        Define a function that can register all logs thrown from the sync method.

        Parameter: message

      • handler

        • onSuccess

          Define a function that will perform an action when sync is successfull

        • onError

          Define a function that will perform an action when sync fails.


      dbInstance.open({//config  goes here})
      .onSuccess(function(ex){
      DB = ex.result;
       })
      .onError(function(ex){
      //Error handler goes in here 
      })
      .onUpgrade(function(ex){
      //do something each time the onUpgrade is initialized
      });

      Note : onUpgrade function only initializes when creating the DB for the first time or when the version number of the DB changes.

  • # transaction.insert(DATA,...)

    • Arguments

      • {ARRAY|OBJECT} DATA (required)

    • Mode: write or writeonlyy

    • jQL: -insert -data -table_name

    • Usage

      Column name must be defined when using OBJECT, while for ARRAY the data must follow according to their column position in the table. When inserting data in ARRAY format, data in each arraylist will be Mapped to the table column according to their index. Fields can be ommited only if they have a default value.
      [{fullName:"Foo Bar",email:"foo@bar.com"},{fullName:"Bar Foo",email:"bar@foo.com"}]
      [[1,"Foo Bar","foo@bar.com"],[2,"Bar Foo","bar@foo.com"]] 
      //Below example inserts data into our table
      DB
      .transaction(tblName,"writeonly")
      .onSuccess(function(ins){
       var res = ins.result;
      res
      .insert(data)
      .execute()
      .onSuccess(function(e){
      //e = EVENT Object 
      })
      .onError(function(e){ 
      //Handle Failed Insert here
      });
      })
      .onError(function(){ 
      //Handle Failed Transaction here
      })

  • # transaction.update(DATA,CLAUSE)

    • Arguments

      • {OBJECT|STRING} DATA (required)
      • {STRING|OBJECT} CLAUSE (optional)

    • Mode: write or writeonly

    • jQL: -update -TBL_NAME -records -where

    • Usage

      • OBJECT : {fullName:"Foo Boo",email:"foo@boo.com"}
      • PARAM : fullName="Foo Boo",email="foo@boo.com"
      //Below example updates our table
      db
      .transaction(TBL_NAME,'writeonly')
      .onSuccess(function(upd){
      upd
      .result
      .update(DATA,WHERE) 
      // requires two parameter DATA to update and which row to update using WHERE
      .execute()
      .onSuccess(function(e){
      result = e;
      })
      .onError(function(e){
      //Update Error Handler
      })})
      .onError(function(e){
      //Transaction Error Handler
      });

  • # transaction.delete(condition)

    • Arguments

      • {OBJECT|STRING} condition

    • Mode: write or writeonly

    • Usage

      db
      .transaction(table,'writeonly')

          del
          .result

        .execute()
         .onSuccess(function(e){
         //success Handler
       })
        .onError(function(e){
      //delete Transaction Error Handler
       })

      //DB Error Handler
       })

  • # transaction.select(STATEMENT)

    • Arguments

      • {STRING} STATEMENT (required)

      • column types

        • * To select all columns
        • -columnName : Select a particular column set from the table ( Seperate multiple columns with a comma)
        • -columnName as user-defined : select columns and return as user-defined Name (eg : -fullName as fName)

    • Mode: read or readonly

    • jQL: -select -columns -table_name [-where [-like]] [-limit]

    • Using CLAUSE in Select

      • WHERE :

        eg : where(id === 1)

      • -LIKE : -LIKE:expression

        eg where(fullName LIKE:foo)

      • Join Clause :

        List of Join Clause to use

        • INNER
        • LEFT
        • RIGHT
        • OUTER
      • Limit:

        eg: limit(0,10)

      //Below example selects record from 1 table
      DB
      .transaction(TBL_NAME)
      .select('name,email,password,id')
      .execute()
      .onError(errorHandler)
      .onSuccess(function(e){
      console.log("Success Result:",e.getResult())
      });
      //Below example selects record from 2 table using the join CLAUSE
      DB
      .transaction([TBL_1,TBL_2])
      .select('TBL_1.COL_NAME,TBL_2.COL_COURSE')
      .join('inner:TBL_2')
      .on('TBL_1.course = TBL_2.id')
      .execute()
      .onError(errorHandler)
      .onSuccess(function(e){
      console.log("Success Result:",e.getResult())
      });
      //using WHERE and LIKE CLAUSE
      DB
      .transaction(TBL_NAME)
      .select('name,email,password,id')
      .where("COL_NAME like:foo")
      .execute()
      .onError(errorHandler)
      .onSuccess(function(e){
      //e.getResult(); displays our query result
      });
      //using Limit Clause
      DB
      .transaction(TBL_NAME)
      .select('COL_1,COL_2,COL_3')
      .where("COL_NAME like:foo")
      .limit('0,10')
      .execute()
      .onError(errorHandler)
      .onSuccess(function(e){
      //console.log("Success Result:",e.getResult())
      });
Note : If "MODE" is undefined transaction is treated as read_only, for insert we re using the writeonly mode.

# DB.synchronize()

  • return: Sync Instance

  • jQl:-sync [optional [-tableName]]

  • Details

    Synchronize changes on the client to the server,synchronizer adapter can compare and check for changes between client and server if the server have the latest changes, the client will be updated with the server records else changes will be pushed to server for update

    DB
    .synchronize()
    .Entity([TBL_1]) // Array of TABLES to sync
    .configSync({}) //Configuration used in sync method
    .processEntity()

    This configurations can also be define when creating the database and it's extensible. Please look at Create a database
